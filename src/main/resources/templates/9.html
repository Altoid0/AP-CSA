Unit 9: Inheritance

Inheritance is when there are two classes that share functionalities. Inheritance is used when you want to create a new class, but want to use pre-existing code from another class. In this way, you are able to use methods from the existing parent super class in the subclass that is created.

Class Hierarchy

Class Hierarchy is the relationship of inheritance between classes, also known as the inheritance tree. The highest level class is known as the super class, and it is important to note that this is the Object Class. The farther away you get from this super class, the more exclusive the class behavior and method calls become.

Here we can see a few different cases of how Inheritance can be used. In Single Inheritance, a single class with extend another class and gain access to the super class's methods. In Multi Level Inheritance, there is a grandchild Class C that extends Class B, which itself extends Class A. In Hierarchical Inheritance, two different classes can extend from one class, but cannot access each other's methods, only the methods from the class that they inherit. Finally, in Multiple Inheritance, a class can inherit methods from more than one parent class, as shown in the diagram where Class C inherits from Class A and Class B.

Subclass and Superclass

The Subclass is the class that extends another class, or inherits methods from another class. The Superclass is the class that is used by the subclass. An important note is that the subclass with inherit everything from the superclass, including: classes, methods, and fields.

Key Words to Know

Extends

In order to show that a class is a subclass of another, we use the extends keyword. The following is an example of how this is used where Output is the subclass, and ConsoleProgram is the superclass.

    public class Output extends ConsoleProgram
    
    {
    
      // put logic code here
    
    }

Super

If we wanted to call the constructor in the superclass from the subclass, we can use the super keyword. The following example shows how we can call the superclass constructor in the subclass Rectangle.

    public class Square extends Rectangle
    
    {
    
      public Square(int length)
      
      {
          super(sideLen, sideLen);
      
      }
    
    }

Subclass Functionality

There are a great many things that can be done from the subclass, as it inherits the private and public variables and methods from the superclass.

  • Subclasses inherit all of the public and protected instance methods of the parent class.
  • Subclasses inherit all of the public and protected instance and class variables.
  • Subclasses can have their own instance variables.
  • Subclasses can have their own static and instance methods.
  • Subclasses can override the parent class's methods.
  • Subclasses can contain constructors that directly invoke the parent class's constructor using the super keyword.
