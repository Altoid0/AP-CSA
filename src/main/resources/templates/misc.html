Math.random

.random is a method from the java.lang.Math class which lets users generate random numbers of type double, by default [0,1). This alone is not very useful however through manipulation we can generate more meaningful numbers.

Code Range Example Values
Math.random() 0 ≤ r < 1 0.123, 0.555, 0.999
Math.random() * 6 0 ≤ r < 6 0.123, 1.000, 5.999
(int) (Math.random() * 6) 0 ≤ r < 6 0, 3, 5
(int) (Math.random() * 6) + 1 0 ≤ r < 7 1, 3, 6

Overall trend: (int) (Math.random() * range) + first number

Example 1: Mr. M wants to randomly choose students from his class to answer questions. If Mr. M has 36 students, each with their own unique number starting from 1, write a Math.random() statement that will select a student.

Answer: (int) (Math.random() * 36) + 1

Example 2: Student #1 dropped out of the class. Please alter the statement to reflect this change.

Answer: (int) (Math.random() * 35) + 2

                We have to decrease the range by 1 because if we are starting at 2 and have a range of 36 we'll end up with 37 which is 1 greater than the number of students in the class

Obscure Object Creation

Sometimes Usually the AP test doesn't do things in an easily understandable or orthodox way. One of the things you have to watch out for are weird object creations inside of methods. In the below example we call a method and rather than creating the object before hand and passing it in, we created it in the method call itself. Both will accomplish the same exact goal but one is more recognizable than the other.

methodcall(new ObjectName(10, “bob”))

Printing Anomalies

When using System.out.print() the order in which stuff is printed matters when adding numbers and printing the sum

    int x = 10;
    int y = 2;
    System.out.println(x + y + “concatenated string”);
    System.out.println(“concatenated string” + x + y);

Returns the following:

12concatenated string

concatenated string102
